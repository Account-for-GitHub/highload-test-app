## Highload-test Application

Это несложное приложение может использоваться для проверки работы балансировщиков нагрузки
или для нагрузочного тестирования микросервисов.

Оно выполняет указанное в конфигурации количество асинхронных запросов на указанный URL. Количество асинхронных запросов
ограничено только ресурсами RAM и CPU системы. В этой версии приложения есть возможность выполнять только запросы POST, 
но предусмотрена возможность расширения функционала добавлением любых других типов запросов, например gRPC или GraphQL.

### Requirements

Для работы приложения необходимо:

* PHP 8.1
* Composer
* Docker
* Для тестов нужен любой web-сервер работающий по адресу: http://localhost:80

### Installation

Для начала работы приложения нужно выполнить установку библиотек composer:

    composer install

Установить сервер MySQL через Docker:

    docker compose up

Выполнить миграции:

    php app/console/migrate.php

### Launch

Для запуска приложения надо выполнить консольную команду:

    ./highload_test

### Reports

В приложении есть возможность генерации файлов содержащих данные обо всех выполненных запросах.
В этой версии приложения есть два формата таких файлов, CSV и HTML, но предусмотрена возможность добавления любых
других форматов.

Для генерации CSV файлов надо выполнить команду:

    php app/console/reports.php

или

    php app/console/reports.php --format="csv"

CSV файлы добавляются в директорию:

    reports/csv

Для генерации HTML файлов надо выполнить команду:

    php app/console/reports.php --format="html"

HTML файлы добавляются в директорию:

    reports/html

### Config

Конфигурация выполняемых highload-тестом запросов указывается в файле:

    app/config/highload-test-config.json

```JSON
{
  "url": "http://localhost",
  "quantity": 3,
  "request": {
    "key_1": "value 1",
    "key_2": "value 2"
  }
}
```

Но ещё есть возможность передать настройки highload-теста через аргументы консольной команды при запуске приложения:

    ./highload_test --url="http://localhost" --quantity="10"

### Database

В проекте есть docker-compose файл с настроенной базой данных MySQL, но при необходимости можно выбрать и другую бд, 
подойдёт любая база данных которую поддерживает Eloquent.
Данные для подключения к бд надо указать в файле:

    app/database/Database.php

```PHP
protected const CONNECTION_CONFIG = [
    'driver' => 'mysql',
    'host' => 'localhost:3306',
    'database' => 'mysql_database',
    'username' => 'root',
    'password' => 'root',
    'charset' => 'utf8',
    'collation' => 'utf8_unicode_ci',
    'prefix' => '',
];
```

### Logs

Логи находятся в директории:

    logs

В логи добавляется различная информация о работе приложения:

* app.log - Общая информация о работе приложения
* error.log - Ошибки выполнения запросов
* response.log - Информация о выполненных запросах
